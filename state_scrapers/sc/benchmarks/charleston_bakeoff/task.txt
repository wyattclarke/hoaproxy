## Available tools

- `hoaware.discovery.probe(lead)` — fetches an HOA homepage, harvests PDF links, verifies them, and banks them to GCS. See hoaware/discovery/probe.py for the API.
- `hoaware.discovery.Lead(name=, state="SC", county="Charleston", city=, website=, source=, source_url=)` — the input dataclass.
- `python -m hoaware.discovery probe-batch leads.jsonl` — batch wrapper around probe(); reads a JSONL of Lead dicts, one per line. Has a 5-min per-HOA timeout (HOA_PROBE_TIMEOUT).
- `hoaware.discovery.bank.bank_hoa(...)` — direct bank API if you want to bank docs without going through probe.
- Serper API: `HOA_SERPER_API_KEY` env var; $0.001/query for Google search. Use for finding HOA websites.
- Existing aggregator scrapers under `hoaware/discovery/sources/` (NC) — read them for patterns, but they don't cover SC. You'll need to find SC-specific sources.
- You may write Python scripts, install packages, scrape websites, anything.

## What counts as banked

Each banked HOA = one manifest.json under `gs://hoaproxy-bank/v1/SC/charleston/<hoa-slug>/manifest.json`.
Each banked PDF = `gs://hoaproxy-bank/v1/SC/charleston/<hoa-slug>/doc-<sha>/original.pdf`.

Count via:
  gsutil ls "gs://hoaproxy-bank/v1/SC/**/manifest.json" 2>/dev/null | wc -l
  gsutil ls -r gs://hoaproxy-bank/v1/SC/ 2>/dev/null | grep "original.pdf" | wc -l
